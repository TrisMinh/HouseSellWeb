# Rule: Planning & Approval First

## Quy tắc bắt buộc

1. Luôn tạo plan trước khi bắt đầu code.
2. Từ plan, tạo task list chi tiết nhất có thể theo từng bước hành động.
3. Trình plan và task cho user xác nhận.
4. Chỉ được code sau khi user đồng ý.
5. Task phải chia nhỏ (atomic) và mỗi task phải gán skill phù hợp.

## Cấu trúc tối thiểu

- `plans/<plan-id>/plan.md`
- `tasks/<plan-id>/tasks.md`

## Tiêu chí task chi tiết

- Có mục tiêu rõ cho từng task.
- Có đầu vào/đầu ra.
- Có tiêu chí hoàn thành.
- Có thứ tự phụ thuộc.
- Có rủi ro chính và cách kiểm tra.
- Có skill dự kiến dùng cho task đó.
